# B02 Supplemental Control-Plane Experiments

Generated by `run_control_plane_supplements.py`.

These CPU-only experiments complement the T4 vLLM measurements. They are intended for paper-writing evidence, not as a replacement for real serving measurements.

## Files

- `admission_ablation.csv`: compares random-K, LRU-K, coverage-K, and demand-aware top-K under high/medium/low prefix locality.
- `staleness_validation.csv`: injects expired leases, wrong epochs, evicted resources, and model mismatches; owner-side validation must force fallback and keep unsafe reuse at zero.
- `rate_controlled_event_driven.csv`: token-bucket event dissemination versus periodic N*K*f reporting.
- `supplemental_key_findings.csv`: compact paper-facing findings.

## Interpretation Rules

- Admission rows measure interface value under a bounded advertised state budget. The strongest claim is relative: demand-aware top-K preserves more exact-affinity value than simple truncation when locality exists.
- Staleness rows are correctness/protocol evidence. They show fallback behavior, not latency improvement.
- Event-driven rows are control-plane model evidence. They should be described as a rate-budget simulation unless repeated with a live metadata transport.
